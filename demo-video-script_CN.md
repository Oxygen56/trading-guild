# Demo 视频脚本 — Casper 自主交易公会

> **目标时长：** 4 分钟（范围 3–5 分钟）
> **语言：** 中文解说（可配英文字幕）
> **录制：** 1080p 30fps 屏幕录制 + 配音
> **导出：** MP4 (H.264)，1080p，立体声

---

## 视频结构（6 段）

```
[0:00–0:25]  第 1 段：钩子 & 问题引入
[0:25–1:00]  第 2 段：面板 & 架构概览
[1:00–2:15]  第 3 段：策略 → 风控 → 执行（成功路径）
[2:15–3:00]  第 4 段：风险拒绝 & Agent 经济
[3:00–3:45]  第 5 段：数据分析 & 透明性
[3:45–4:00]  第 6 段：收尾 & 行动号召
```

---

## 第 1 段：钩子 & 问题引入（0:00–0:25）

### 画面
- 从黑色淡入
- 分屏画面：左侧 = 盯着 6 块屏幕的疲惫人类交易员；右侧 = Trading Guild Logo，三个 Agent 头像（紫/Athena、金/Guardian、青/Mercury）脉动活跃
- 文字叠层逐词出现

### 解说
> "DeFi 交易很累。你 7×24 小时盯盘，每笔交易都反复犹豫。你同时是策略师、风控师、交易员——一个人扛所有角色。
>
> 如果不用这样呢？
>
> 如果 AI Agent 能搞定一切——彼此协作、相互支付、自主交易——而且每一个决策都在链上可验证？
>
> 这就是 Casper 自主交易公会。"

### 转场
- 分屏收拢为 Trading Guild Logo
- 放大切入总览面板

---

## 第 2 段：面板 & 架构概览（0:25–1:00）

### 画面
- 展示公会总览面板（深色主题，翡翠绿强调色）
- 鼠标跟随解说移动指向各 UI 元素
- 约 0:40 处叠层显示架构动画图

### 解说
> "这是公会总览面板。一眼就能看到金库——目前管理着 50,000 CSPR 的资产。三个 AI Agent 在线并活跃。
>
> Athena——策略 Agent——扫描市场、提出交易。
> Guardian——风控 Agent——每笔策略执行前先审核。
> Mercury——执行 Agent——在链上执行已批准的交易。
>
> 它们不只是聊天机器人。它们是自主的经济实体——各自有钱包、各自有定价、各自的信誉都压在线上。
>
> 来看看它们怎么协作。"

### 关键 UI 展示
1. 金库概览数据卡片（TVL、24h 盈亏、活跃策略数、Agent 数量）
2. Agent 列表网格（3 张 Agent 卡片，含状态指示）
3. 最近动态时间线（滚动交易历史）

### 转场
- 点击侧边栏"对话终端"
- 画面切换至 AI 对话终端

---

## 第 3 段：策略 → 风控 → 执行（1:00–2:15）

### 画面
- 对话终端：用户输入自然语言指令
- 展示消息发送过程
- 三栏分屏依次展示各 Agent 的处理过程
- 底部交易流程图动画

### 解说
> "这是核心流程。我在终端输入一行简单的指令：
>
> '找出 CSPR.trade 上滑点在 5% 以内的最优收益机会。'
>
> Athena——策略 Agent——扫描 CSPR.trade 的流动性池。它找到了 CSPR-USDT 池：TVL 210 万美元，年化 35.2%。Athena 生成一个 swap 策略——5,000 CSPR 换成 USDT，扣除手续费后预期收益 2.3%。
>
> 但在资金动用之前，Athena 必须先获得批准。它把策略发给 Guardian，风控 Agent——并通过 x402 支付审核费。0.005 CSPR，链上结算。没有 API Key，没有订阅——就是一笔直接的 Agent 对 Agent 支付。
>
> Guardian 执行 6 维风险检查：代币安全性、流动性深度、滑点模拟、池集中度、历史波动率、近期活跃度。评分：8 分（满分 100，越低越好）。低风险。批准。
>
> Athena 现在向 Mercury——执行 Agent——支付 0.01 CSPR 来执行交易。Mercury 通过 MCP 协议调用 CSPR.trade 的 DEX，提交 swap，返回交易哈希。Casper Testnet 上已确认。
>
> 从自然语言到链上交易——除了最初那行指令，全程无需人类介入。"

### 关键 UI 展示
1. 对话终端：用户输入气泡和 Agent 回复气泡（按 Agent 颜色区分）
2. Agent 状态栏：展示状态转换（空闲 → 处理中 → 完成）
3. 策略卡片展开显示完整审核轨迹
4. 链上交易链接出现，附带 Casper Testnet 浏览器 URL

### 转场
- 画面聚焦支付账本
- 叠层文字："如果策略太危险呢？"

---

## 第 4 段：风险拒绝 & Agent 经济（2:15–3:00）

### 画面
- 另一个终端输入
- Guardian 的风险评估面板——红色指标，评分突破 50
- 支付账本显示审核费已赚取但策略被拒绝
- 金库余额不变（绿色边框脉冲高亮）

### 解说
> "不是每笔策略都能通过。这里，我让 Athena 分析一个新上线的代币——Guardian 发现了危险信号。
>
> 这个代币卖出税高达 90%，流动性池 2 小时前才创建，部署者钱包通过混币器注资。Guardian 打出 51 分——超过了拒绝阈值。
>
> 结果：策略被拦截。0.005 CSPR 的审核费照样支付——Guardian 尽责了，该赚的钱一分不少——但公会资金零风险。
>
> 这就是核心创新：Agent 有切身利益。Guardian 帮公会避免亏损交易时赚得更多。Mercury 按成功执行比例赚取费用。每个 Agent 的经济激励都与公会整体收益对齐。"

### 关键 UI 展示
1. 策略卡片显示 REJECTED（已拒绝）状态徽章（红色）
2. 风控审核明细面板（6 个维度及各自评分）
3. 支付账本条："审核：0.005 CSPR | 结果：已拒绝"
4. 金库余额：未变动（绿色确认）

### 转场
- 点击侧边栏"数据分析"
- 面板切换至数据分析页面

---

## 第 5 段：数据分析 & 透明性（3:00–3:45）

### 画面
- 盈亏图动画（Recharts 组合图——盈利柱绿色，亏损柱红色，累计线翡翠绿）
- 滚动至 Agent 绩效表
- 高亮分配轮次

### 解说
> "每一个决策、每一笔支付、每一次交易——在数据分析页面上完全透明。
>
> 盈亏图追踪公会随时间推移的表现。你能清楚看到哪些策略赚了钱，哪些亏了钱。
>
> Agent 绩效表展示每个 Agent 的业绩记录：提出了多少策略、批准了多少、执行了多少——最重要的是，创造了多少利润。这不只是数据面板——这是一个信誉系统。表现更好的 Agent 会获得更多信任和更多资金分配。
>
> 利润分配是算法驱动的。公会平仓盈利头寸时，收益回流金库，按每个 Agent 的贡献权重按比例分配。全程链上，全程可审计。
>
> 这就是全 Agent 公司的样子。"

### 关键 UI 展示
1. 盈亏图：盈利柱、亏损柱、累计线
2. Agent 绩效表：可排序列（利润、批准率、策略数、分配份额）
3. 分配历史：逐轮明细

### 转场
- 淡出至黑色
- Trading Guild Logo 淡入

## 录制检查清单

### 录制前
- [ ] 所有 Agent 运行正常（`npm run demo` 成功）
- [ ] 前端运行中（`cd frontend && npm run dev`）
- [ ] Mock 模式开启保证干净演示（`NEXT_PUBLIC_CASPER_USE_MOCK=true`）
- [ ] 浏览器窗口调整为 1920×1080
- [ ] 浏览器扩展隐藏，书签栏关闭
- [ ] 终端窗口隐藏（或移出屏幕）
- [ ] 深色模式确认（macOS + 浏览器）

## 解说要点

- **节奏：** 沉稳自信，不急不赶。这是技术演示，不是路演。
- **停顿：** 每次主要 UI 切换后稍作停顿——让观众消化所看内容。
- **创新点强调：** 提到 x402 支付或 Agent 间经济时，放慢语速。这些是新颖概念。
- **发音提示（英文术语）：**
  - "Casper" = 读作 "开斯珀"
  - "x402" = 读作 "X 四零二"
  - "Athena" = 雅典娜 / "Guardian" = 守护者 / "Mercury" = 墨丘利（或直接用英文名）
  - "Odra" = 读作 "欧德拉"
